# Formulas and Reference Tables — LLM Database Theory

## Core Mathematical Correspondences

### Attention as KNN Query

Standard scaled dot-product attention:

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) · V
```

KNN graph-walk interpretation:
- Q = query vector (token seeking context)
- K = key matrix (indexed node features)
- V = value matrix (node contents / edge targets)
- `QK^T / sqrt(d_k)` = similarity scores (edge weights)
- `softmax(...)` = normalised attention weights over neighbour nodes
- `... · V` = weighted aggregation of target-node values

These are the same operation: attention **is** a weighted KNN retrieval over
the key-value graph encoded in the weight matrices.

### FFN Layer as Key-Value Store

A two-layer FFN with activation σ:

```
FFN(x) = W_down · σ(W_up · x)
```

- `W_up` (d_model × d_ff): key matrix — rows are feature detectors
- `σ(W_up · x)`: which features fire for this input (gate activation)
- `W_down` (d_ff × d_model): value matrix — rows are associated output vectors
- Each column of `W_up^T` paired with corresponding row of `W_down` = one edge entry

Number of edge entries per FFN layer: d_ff (the hidden dimension, e.g. 11,008 for Llama 3 8B)

### Layer Depth → Graph Hierarchy

| Transformer depth | Graph DB analogue | Knowledge type |
|---|---|---|
| Lower layers (L1–L8) | Shallow index (token surface) | Syntax, morphology, local co-occurrence |
| Middle layers (L9–L20) | Primary KG (entity-relation-target) | Factual assertions, named entities |
| Upper layers (L21–L32+) | Context layer (episodic / dynamic) | Discourse context, in-prompt facts |

LARQL's DESCRIBE output shows layer numbers (L_n) per edge; upper-layer
entries are more specific to the current context; lower-layer entries are
more stable across queries.

### Categorical Structure

Self-attention linear component formalised as a parametric endofunctor F
in Para(Vect):

```
F : Vect → Vect
F_θ(X) = MultiHead(X; W_Q, W_K, W_V)  (linear part only)
```

Stacked attention layers form the free monad T_F:

```
T_F(X) = X + F(X) + F²(X) + ... + F^L(X)  (via residual connections)
```

This means: the composition of L transformer blocks is the free monad
generated by the single-layer endofunctor F. Properties of F propagate
algebraically through the composition.

## LARQL Operation ↔ SQL Analogue

| LQL operation | SQL analogue | Database right triggered |
|---|---|---|
| `DESCRIBE "entity"` | `SELECT * FROM edges WHERE subject = 'entity'` | Read access to DB contents |
| `WALK "entity" DEPTH 3` | Multi-hop graph traversal / recursive CTE | Read access to relational structure |
| `INSERT INTO EDGES ...` | `INSERT INTO edges (s, p, o) VALUES (...)` | Write access — confirms mutability |
| `COMPILE CURRENT INTO MODEL ...` | `EXPORT DATABASE AS ...` | Full database export/serialisation |
| Inference / INFER | `SELECT target FROM edges WHERE ... ORDER BY score DESC LIMIT 1` | Query execution (read + compute) |
