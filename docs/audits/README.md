# Footnote-to-Citation Audit

`footnote-citation-audit.csv` is generated by `docs/scripts/audit_footnote_links.py`. It is a
**read-only report** — no corpus document is modified by generating it. See
[the design spec](../superpowers/specs/2026-07-23-footnote-citation-audit-design.md) for the
full rationale, and [the implementation plan](../superpowers/plans/2026-07-23-footnote-citation-audit-plan.md)
for how it was built.

Each row is a candidate footnote-number marker matched positionally to a works-cited entry, with
a confidence tier (`High` / `Medium` / `Needs manual triage`) and a `flags` column explaining why.
Regenerate with `python3 docs/scripts/audit_footnote_links.py` any time the corpus changes.

**Current run:** 7,667 candidates — 6,129 High, 1,025 Medium, 513 Needs manual triage — across 95
documents. Verified against real content by a full human-review pass (all 40 `restart_detected`
rows, 30 `High`, 20 `Medium`, and a stratified 30-row sample of `Needs manual triage`).

**Two known, honestly-labeled data-quality caveats found during that review** (corpus defects the
audit correctly surfaces, not audit bugs — see design spec §9, "explicitly sequenced follow-on
work," for the broader cleanup this belongs to):

1. **~32 `High`-confidence rows have thin entry text** (`", accessed <date>, "` plus a bare
   domain like `sec.gov`/`irs.gov`) because the *source* works-cited entry itself is malformed —
   a citation with a real, resolvable link but no descriptive title. The `High` tier is correct
   (the link corroboration is real), but a human triager will need to open the source document to
   see what's actually cited.
2. **`contractual-architectures-for-ip-and-cooperatives.html`'s works-cited list contains ~3
   entries that are LLM-prompt-fragment text**, not real citations (corpus pollution from an
   earlier authoring/conversion step). The audit routes these to `Needs manual triage`/`Medium`
   via the missing-link flag, but doesn't specifically distinguish "this isn't a citation at all"
   from an ordinary missing link — a human triager will recognize them immediately on inspection.

**This report identifies candidates for a future footnote-to-citation linking pass — it does not
perform one.** Any mechanical rewrite of the corpus's 446 files based on this report's `High` rows
is a separate, not-yet-designed project (design spec §9).
